# Unit 1

## Software Testing
**Software Testing Definition:**
> Software testing is the process of evaluating and verifying that a software product or application functions correctly, securely and efficiently according to its specific requirements.

### Objective of Testing:
> The main objective of testing is to identify defects (bugs) and verify that the software meets specified requirements and works correctly before it is delivered to users.

1. **Detect defects and errors:**
    - The foremost objective of software testing is to identify errors, defects, or faults present in the software before it is released to the users. 
    - Detecting defects early reduces the cost and effort required to fix them.
    - **Example:** If a login page accepts incorrect passwords, testing helps identify this issue before deployment.

2. **Verify Software Requirements**
    - Verification ensures that the software has been developed according to the specified functional and non-functional requirements.
    - Testing checks whether every feature behaves exactly as mentioned in the Software Requirement Specification (SRS).

3. **To Validate User Requirements**
    - Validation ensures that the developed software actually fulfills the customer's needs and expectations.
    - Even if software meets all technical specifications, it is considered unsuccessful if it does not satisfy the end users.

4. **To Improve Software Quality**
   - Testing helps improve various quality attributes of software, such as:
      - Reliability
      - Performance
      - Security
      - Usability
      - Efficiency
      - Compatibility
      - Maintainability
    - A well-tested software product provides a better user experience and performs consistently under different conditions.


5. **To Reduce Risks**
    - Software failures can result in financial losses, security breaches, data corruption, or even loss of human life in critical systems.
    - Testing minimizes these risks by identifying potential problems before the software is deployed.


6. **To Ensure Security**
   - Testing identifies security vulnerabilities such as:
     - Unauthorized access.
     - Weak authentication
     - SQL Injection
     - Cross-Site Scripting (XSS)
     - Data leakage
   - Security testing protects sensitive user information and maintains trust.


7. **To Evaluate Performance**
    - Performance testing checks whether the software performs efficiently under different workloads.
    - It evaluates:
      - Response time
      - Speed
      - Scalability
      - Stability
      - Resource utilization
    - **Example:** An online shopping website should remain responsive during festival sales with thousands of simultaneous users.

In short, we can say, the objective of testing is to find defects, verify that the software meets specified requirements, and ensure that it functions correctly and reliably to satisfy user needs.

## Error
**Definition:**
> An Error is a human mistake made during software development.It occurs when a developer, tester, or analyst makes an incorrect decision or misunderstanding.

**Example:**
- Writing wrong logic in code.
- A developer writes a - b instead of a / b.
- A requirement is misunderstood by the analyst.
- A tester creates an incorrect test case.

**Key points:**
- Created by humans.
- Occurs during software development.
- May lead to faults in the software.

## Fault
**Definition:**
> A Fault is an incorrect piece of code, design, or documentation that results from an error.A fault is also known as a software fault or coding fault.

**Example:**
- Suppose the requirement is to create a user registration page, but the code is written to create a login page.

**Key Points:**
- Introduced due to human errors.
- Exists in the software code or design.
- Can cause failures when executed.

## Defect
**Definition:**
> A Defect is a deviation of the software from its expected behavior or specified requirements. It is identified during software testing when the actual result differs from the expected result.

**Example:**

- *Requirement:*
    - The login page should allow only registered users.

- *Actual Behavior:*
    - The login page allows both registered and unregistered users.
> This mismatch is called a Defect.

**Key Points:**
- Found during testing.
- Indicates mismatches with requirements.
- Usually reported by testers.

## Bug
**Definition:**
> A Bug is an informal term used to describe a defect or fault in the software. Once a tester reports a defect, developers commonly refer to it as a bug while fixing it.In practice, the terms bug and defect are often used interchangeably.

**Example**
- If clicking the "Submit" button crashes the application, developers refer to this issue as a bug.
- If a tester finds a bug in the application, they report it to the developers.

**Key Points**
- Informal term.
- Usually reported by testers.
- Fixed by developers.

## Failure
**Definition:**
> A Failure occurs when the software is executed and does not perform its intended function or produces an incorrect result. A failure can be caused by errors, faults, or defects.

**Example**
-  banking application should transfer ₹5,000 but transfers ₹50,000 instead.
- The incorrect output observed by the user is a failure.

Key Points
Occurs during software execution.
Visible to users or testers.
Caused by one or more faults.
