# Unit 1

## Software Testing
**Software Testing Definition:**
> Software testing is the process of evaluating and verifying that a software product or application functions correctly, securely and efficiently according to its specific requirements.

### Objective of Testing:
> The main objective of testing is to identify defects (bugs) and verify that the software meets specified requirements and works correctly before it is delivered to users.

1. **Detect defects and errors:**
    - The foremost objective of software testing is to identify errors, defects, or faults present in the software before it is released to the users. 
    - Detecting defects early reduces the cost and effort required to fix them.
    - **Example:** If a login page accepts incorrect passwords, testing helps identify this issue before deployment.

2. **Verify Software Requirements**
    - Verification ensures that the software has been developed according to the specified functional and non-functional requirements.
    - Testing checks whether every feature behaves exactly as mentioned in the Software Requirement Specification (SRS).

3. **To Validate User Requirements**
    - Validation ensures that the developed software actually fulfills the customer's needs and expectations.
    - Even if software meets all technical specifications, it is considered unsuccessful if it does not satisfy the end users.

4. **To Improve Software Quality**
   - Testing helps improve various quality attributes of software, such as:
      - Reliability
      - Performance
      - Security
      - Usability
      - Efficiency
      - Compatibility
      - Maintainability
    - A well-tested software product provides a better user experience and performs consistently under different conditions.


5. **To Reduce Risks**
    - Software failures can result in financial losses, security breaches, data corruption, or even loss of human life in critical systems.
    - Testing minimizes these risks by identifying potential problems before the software is deployed.


6. **To Ensure Security**
   - Testing identifies security vulnerabilities such as:
     - Unauthorized access.
     - Weak authentication
     - SQL Injection
     - Cross-Site Scripting (XSS)
     - Data leakage
   - Security testing protects sensitive user information and maintains trust.


7. **To Evaluate Performance**
    - Performance testing checks whether the software performs efficiently under different workloads.
    - It evaluates:
      - Response time
      - Speed
      - Scalability
      - Stability
      - Resource utilization
    - **Example:** An online shopping website should remain responsive during festival sales with thousands of simultaneous users.

In short, we can say, the objective of testing is to find defects, verify that the software meets specified requirements, and ensure that it functions correctly and reliably to satisfy user needs.
