# Unit 1 : Part 2

## Software Development Life Cycle (SDLC):
**Definition:**
> The Software Development Life Cycle (SDLC) is a systematic process for planning, designing, coding, testing, and deploying software applications.It provides a structured framework that helps software teams produce high-quality software that meets customer requirements within the specified time and budget.

The Software Development Life Cycle (SDLC) typically consists of 6 to 7 essential phases:
1. **Planning**
2. **Analysis**
3. **Design**
4. **Coding**
5. **Testing**
6. **Deployment**
7. **Maintenance**

### 1. **Planning:**
> This stage determines whether the project is technically, financially, and operationally feasible.Planning determines the project's scope, schedule, budget, resources, risks, and development methodology.

**Activities:**
- Feasibility analysis.
- Cost estimation.
- Time estimation
- Resource allocation
- Risk analysis
- Project scheduling

**Key Roles:**
- Project manager
- Senior Engineers, 
- Stakeholders

**Outputs:**
- Project plan
- Risk Management Plan

### 2. **Analysis:**
> During the analysis phase, the development team collects and analyzes information on the project’s requirements. Analysis enables the team to advance the work they started in the planning phase, moving from a high-level idea to a practical implementation plan.   

**Activities:**
- Gather customer requirements.
- Conduct feasibility studies.
- Identify functional and non-functional requirements.
- Prepare the Software Requirement Specification(SRS) document.
- Obtain customer approval.

**Key Roles:**
- Business Analysts
- Product Owners

**Outputs:**
- Software Requirement Specification (SRS)

### 3. **Design:**
> In this stage, the approved requirements are transformed into a technical blueprint for implementation. **Types of Design:**
- High-level design(HLD): Defines system architecture, technology stack, database design, and major modules.
- Low-level design (LLD): Specifies component logic, APIs, data structures, and workflows.


**Activities:**
- Design software architecture
- Design database
- Design user interface
- Prepare system diagrams
- Select technologies


**Key Roles:**
- Architects
- Senior Software Engineers

**Outputs:**
- Design Document Specification (DDS)

### 4. **Coding:**
> The coding phase, or development phase, is when the team starts writing the code and building the software, based on the SRS, DDS and other guidelines created during the preceding phases. Developers build the software based on the approved design.


**Activities:**
- Write source code
- Implement modules
- Perform unit testing
- Code review
- Version control management

**Key Roles:**
- Frontend Developers
- Backend Developers
- Full Stack Developers

**Outputs:**
- Source code
- Executable application / Executable software modules

### 5. **Testing:**
> The testing phase begins after the development team has developed a functional software product. During this phase, testers identify defects (bugs), verify that the software meets the specified requirements, and improve its overall quality. The primary objective of testing is to detect defects and ensure that the software satisfies all functional and non-functional requirements before deployment.

**Activities:**
- Prepare test cases
- Execute test cases
- Identify defects
- Report bugs
- Perform regression testing

**Types of Testing:**
- *Unit Testing:* Verifies individual components
- *Integration Testing:* Ensures modules work together.
- *System Testing:* Validates the complete system
- *User Acceptance Testing (UAT):* Confirms business requirements are met.

**Key Roles:**
- Senior Testing Engineers
- Testing Engineers


**Outputs:**
- Test reports
- Defect reports
- Quality metrics


### 6. **Deployment:**
> After successful testing, the software is released and installed in the customer's environment for actual use.

**Activities:**
- Install software
- Configure the system
- Train users
- Release the product

**Key Roles:**
- DevOps Engineers
- Release Managers

**Outputs:**
- Live Software System

### 7. **Maintenance:**
> Maintenance is the final phase of SDLC in which the software is continuously monitored, updated, and improved after deployment.This phase usually consumes the largest portion of the software's lifetime.

**Activities:**
- Bug fixes
- Performance tuning
- Updates
- Feature enhancements

**Key Roles:**
- Support Engineers
- Developers

**Outputs:**
- Patches and updates
- Security updates
- Release new versions

*Software Development Life Cycle Models:*
Software Development Models are structured frameworks that guide the planning, execution, and delivery of software projects. They define the sequence of development stages, such as requirements, design, coding, testing, and deployment.
